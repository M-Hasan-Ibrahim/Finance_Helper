Chapter 4

Visual Perception

How the human visual system affects the speed, accuracy, and clarity of visualization.

Why Perception Matters

Users do not read every visual element one at a time. The visual system detects some properties faster and more accurately than others, so visualization design should match human perceptual strengths.

  • Psychophysics: studies how humans perceive physical stimuli.
  • Cognitive psychology: studies thought, memory, and interpretation.
  • Semiotics: studies symbols and how they communicate meaning.

Two-Stage Model of Visual Processing

Stage 1: Parallel low-level processing → Stage 2: Sequential goal-directed processing

Stage 1: Low-Level Parallel Processing

This stage is fast and automatic. It detects color, texture, shape, motion, orientation, and spatial position in parallel, making visual pop-out possible.

Stage 2: Sequential Goal-Directed Processing

This slower, attention-dependent stage handles object recognition, memory, interpretation, symbolic reasoning, and action planning.

Pre-attentive Processing and Pop-Out

Pre-attentive features are detected almost immediately without focused attention, often in less time than a single eye movement.

200 to 250 ms

Common pop-out features include hue, size, length, width, curvature, orientation, motion, intensity, closure, and depth cues.

Visual Search Tasks

  • Target detection: determine whether an item is present.
  • Boundary detection: determine whether elements form groups.
  • Counting: count elements of a particular type.

Hue, Shape, and Conjunction Search

A single feature such as hue or shape can often be detected pre-attentively. A red circle among blue circles or a square among circles is easy to find.

A target defined by both hue and shape requires a conjunction search. Finding a red circle among red squares and blue circles is slower because attention must examine combinations of features.

Single featureFast search
Combined featuresSlower search

Guided Search Principles

  • Set size: more distractors generally increase search time.
  • Presence: an absent target usually takes longer to confirm.
  • Target similarity: a more distinctive target is easier to find.
  • Distractor heterogeneity: varied distractors make search harder.
  • Flanking: nearby distractors can slow search.
  • Asymmetry: searching for A among B may differ from searching for B among A.
  • Guidance: visual cues can direct attention.

Change Blindness and Attention

Change blindness occurs when viewers fail to notice a change because attention is limited. Even large changes can be missed when attention is directed elsewhere.

  • Attentional blink: missing a second target shortly after seeing the first.
  • Selective attention: focusing on one task hides other visible events.
  • Optical illusions: perceived reality differs from physical reality.

Designers should never assume that users will automatically notice every visual change.

Visualization Components

  • Space: where data is placed.
  • Marks: visible objects representing data.
  • Implantation: how marks are placed in space.
  • Visual variables: properties used to encode data.

The main mark types are points, lines, and areas.

Visual Variables

Position, color, length, area, angle, size, shape, brightness, and texture are all visual variables. Their selection determines how accurately users can read the encoded data.

Luminance and Brightness

  • Luminance: the physical amount of light coming from a surface.
  • Brightness: the amount of light a person perceives.

Brightness is not a linear function of luminance. Foreground and background colors therefore need strong luminance contrast for text and marks to remain readable.

Greyscale

Greyscale can encode ordered data, but contrast depends heavily on context, surface orientation, and background. It becomes unreliable when too many levels must be distinguished.

Approximately 10 distinguishable shades

Visual Operations

  • Selective: can targets be selected using the attribute?
  • Associative: can targets be grouped using the attribute?
  • Quantitative: can a numerical value be read from it?
  • Orderable: is an ordering perceptible?
  • Discriminability: how many distinct levels can be distinguished?

Color

The HSV model describes color through three components:

  • Hue: what people normally call color.
  • Saturation: intensity from gray to pure color.
  • Value: lightness or darkness.

Color for Categories

Categorical color should be limited to a manageable number of clearly different colors.

≤ 12 distinct colors

Color for Ordered Data

Ordered values require a perceptually ordered scale. Suitable choices include greyscale, single-hue sequential scales, multi-hue sequential scales, and diverging scales.

The Rainbow Scale Problem

Rainbow color maps create artificial bands and sharp transitions that can look like real patterns.

rainbow scale → false visual boundaries

Color Design Rules

  • Maintain high luminance contrast between foreground and background.
  • Use only a few distinct colors.
  • Prefer red, green, yellow, and blue first for categorical distinctions.
  • Use muted colors for large areas.
  • Check that the design remains understandable for color-blind users.

Accuracy of Visual Variables

People judge some visual encodings more accurately than others:

position and length > area > volume

Length is estimated relatively well, while area and volume are often underestimated. This is why bar charts are generally easier to read accurately than bubble charts or 3D volume charts.

Combining Visual Variables

Separable variables can be read independently, while integral variables interfere with one another and are perceived together.

SeparablePosition and color
More integralWidth and height, or combined color dimensions

Chapter Takeaway

Visual properties are not equally effective. Good visualization design uses perceptual strengths, avoids misleading encodings, limits unnecessary complexity, and chooses visual variables that match both the data and the user's task.
