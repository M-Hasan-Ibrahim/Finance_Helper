Chapter 1

Introduction to Information Visualization

How visual representations transform raw data into insight, understanding, and better decisions.

What is Information Visualization?

Information Visualization is the use of visual representations of data to help people understand, analyze, and communicate information. Its goal is not simply to create images, but to produce insight through discovery, decision-making, and explanation.

Why Visualization is Needed

Modern society produces massive amounts of data from the internet, finance, weather, sensors, logs, images, videos, and many other sources. This creates data overload: people have access to more data than they can easily understand.

The challenge is to transform raw data into useful information. Visualization makes patterns, trends, relationships, and outliers easier to identify.

Why Use Human Vision?

Human vision is the highest-bandwidth sense. It is fast, parallel, and highly effective at pattern recognition. Visual displays also extend memory and cognitive capacity by placing information outside the mind, where it can be inspected and compared.

A well-designed visualization can therefore reveal relationships much faster than tables of numbers alone.

Example: Anscombe's Quartet

Anscombe's Quartet demonstrates that datasets can share the same statistical properties—including mean, variance, correlation, and regression line—while having very different visual patterns. Summary statistics alone can therefore hide important structures.

Same statistics ≠ same data pattern

Visualization matters because it can reveal structures that numerical summaries miss.

Main Definition

The use of computer-supported, interactive visual representations of data to amplify cognition.

In other words, visualization uses interactive visual tools to help people think more effectively.

Three Related Subfields

  • Scientific Visualization: visualizes physical or geometric phenomena, such as airflow, clouds, or stress on a structure.
  • Information Visualization: visualizes abstract data without a direct physical shape, such as tables, networks, or statistics.
  • Visual Analytics: combines visualization with data mining while keeping the human in control of the analysis.

Key Attributes of Visualization

  • Scale: visualization often needs to handle very large datasets.
  • Interactivity: users need to explore data from multiple perspectives.

The InfoVis Pipeline

The visualization process transforms raw data into a useful view for a particular task.

Source Data→ Data Tables→ Visual Abstraction→ Views→ Task

Raw data is structured, mapped to visual elements, and then used by a person to perform a task.

Data Tables

  • Rows represent cases or individual items.
  • Columns represent variables or attributes.
  • Cells contain the value of a variable for a particular case.
f(casei) = ⟨valuei,1, valuei,2, …, valuei,n⟩

Variable Types

  • Nominal: categories with no inherent order, such as gender or color.
  • Ordinal: ordered categories, such as rank or school year.
  • Quantitative: numerical values for which mathematical operations are meaningful, such as age, salary, or temperature.

Metadata

Metadata is descriptive information that explains how data should be interpreted. It can identify a variable's type, the date of collection, units, or logical constraints.

number of home runs ≤ number of at-bats

Metadata matters because a data table alone is often insufficient to explain the data's meaning.

Chapter Takeaway

Information Visualization transforms large and complex data into visual forms that support understanding. It combines human vision, external memory, and interaction to help users discover patterns, compare information, detect outliers, and make better decisions.
