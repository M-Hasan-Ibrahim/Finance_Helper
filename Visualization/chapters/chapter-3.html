Chapter 3

Tasks and Interaction

How visualization helps users analyze data, make sense of information, and interact with complex datasets.

From Data to Task

A visualization is not only about showing data. It must help users perform tasks such as finding values, filtering, comparing, detecting anomalies, and discovering patterns.

Source Data→ Data Tables→ Visual Abstraction→ Views→ Task

The first stages explain how data becomes a visual display. The final stage describes what the user wants to accomplish with it.

Sensemaking

Sensemaking is the process through which people understand data, manage uncertainty, build interpretations, and make decisions. It is iterative rather than linear: users move between collecting data, organizing information, finding evidence, creating hypotheses, and presenting results.

Visualization supports this process by making patterns visible, helping users compare alternatives, and allowing them to refine their understanding step by step.

Low-Level Analytic Tasks

Retrieve Value

Find the attributes of a particular data case—for example, selecting a country to see its income, population, or education level.

Given a case → find its values

Filter

Show only cases that satisfy selected conditions, such as houses within a price range and with a minimum number of rooms.

Show cases where condition = true

Compute Derived Value

Calculate an aggregate or new value from existing data.

average  ·  sum  ·  percentage  ·  difference

Find Extremum

Find the cases with the minimum or maximum value for an attribute.

min(x),   max(x)

Sort

Rank cases according to an ordered metric so high and low values are easier to compare.

x1 ≤ x2 ≤ x3 ≤ …

Determine Range

Identify the span of values in a set of cases.

range = xmax − xmin

Characterize Distribution

Understand common and rare values, concentration, spread, skewness, and the overall shape of a distribution. Histograms, density plots, and box plots commonly support this task.

Find Anomalies

Identify values that differ strongly from the expected pattern, such as outliers, strange peaks, unexpected drops, or unusual relationships.

anomaly = case that differs strongly from expectation

Cluster

Discover groups of similar cases, categories, or patterns in the data.

similar cases → same cluster

Correlate

Determine whether two attributes are related. Scatterplots are commonly used to reveal positive or negative relationships.

x ↑ ⇒ y ↑    or    x ↑ ⇒ y ↓

Summary of Analytic Tasks

Retrieve value Filter Compute derived value Find extremum Sort Determine range Characterize distribution Find anomalies Cluster Correlate

Interaction in Visualization

Interaction allows users to change what they see and how they see it. This becomes particularly important for large or complex datasets that cannot be understood through a single static view.

Select

Mark an item as interesting, such as clicking a point in a scatterplot to highlight it.

Explore

Navigate through the data to reveal other elements or regions.

Reconfigure

Change the arrangement by reordering axes, sorting bars differently, or reorganizing the layout.

Encode

Change the visual representation, such as switching chart types or assigning a different variable to color.

Abstract or Elaborate

Show less or more detail. Zooming out provides a summary, while zooming in exposes individual cases.

abstract = less detail   ·   elaborate = more detail

Filter

Use controls such as sliders or checkboxes to display data conditionally.

Connect

Reveal related items, such as highlighting the neighboring nodes connected to a selected network node.

Shneiderman's Mantra

Overview first, zoom and filter, details on demand.
  1. Overview first: present the global structure of the data.
  2. Zoom and filter: let users focus on the relevant portion.
  3. Details on demand: reveal exact information only when requested.

This avoids overwhelming users with details at the beginning of an analysis.

Dynamic Queries

Dynamic queries let users filter data interactively and see the result immediately. Rather than writing a database query, a person can move sliders and instantly see which cases match.

200,000 ≤ price ≤ 400,000

This makes filtering faster, more visual, and easier to understand.

Response Time

Interaction must be fast enough to preserve the user's thinking process.

0.1 sFeels instantaneous and preserves visual continuity.
1 sThe delay is noticeable, but the interaction still feels connected.
10 sFocus is lost and the cognitive process is interrupted.

Dealing with Big Data

Datasets containing millions or billions of records cannot usually be shown directly. A common solution is aggregation, which combines many cases into a summary representation.

many cases → summary representation

Case Study: Twitter Log Data

Twitter user-event logs can reach terabytes per day. Instead of visualizing every event, systems summarize behavioral sequences such as:

web:home:mentions:stream:avatar:profile_click

The aim is to reveal changes in behavioral patterns over time while preserving interactive performance.

Chapter Takeaway

Visualization design should begin with user tasks. Analytic tasks define what users need to discover, while interaction techniques help them move from overview to focused analysis and details. Effective systems must also provide fast feedback and remain interactive at large data scales.
