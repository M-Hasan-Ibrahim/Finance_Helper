Chapter 5

Graphs and Trees

Visualizing relationships, networks, and hierarchical structures while balancing readability and scale.

Critiquing a Visualization

A visualization should do more than look attractive. A useful critique considers:

  • Chart integrity: does it represent the data faithfully?
  • Users and tasks: who is it for, and which questions should it answer?
  • Visual design: are perception, color, layout, and encodings used effectively?

What is a Graph?

A graph is an abstract mathematical object made of vertices, also called nodes, and edges, also called links.

Graph = vertices + edges

Nodes can represent people, cities, websites, or stations. Edges can represent friendships, roads, hyperlinks, or transport lines.

Ways to Represent a Graph

Adjacency matrixA table whose cells indicate connections between node pairs.
Adjacency listA list containing the neighbors of each node.
Node-link diagramA visual drawing of nodes as marks and edges as lines.

The mathematical graph remains abstract; visualization gives it a concrete visual form.

Types and Properties of Graphs

  • Directed: edges have a direction.
  • Undirected: edges have no direction.
  • Cycles: paths return to their starting node.
  • Degree: the number of edges connected to a node.
  • Weights: numerical values attached to edges.

A road network may be undirected, while a citation network is directed.

What Can Be Modeled as a Graph?

Social networks Train and metro networks Telephone systems The World Wide Web Software call graphs Distribution networks Semantic maps

Graph Drawing

Graph drawing determines where nodes are placed and how edges are routed. Good layouts aim to:

  • minimize edge crossings and total edge length,
  • avoid extremely long edges,
  • keep edge lengths reasonably uniform,
  • minimize bends,
  • use available space efficiently,
  • preserve overall readability.

Graph Layout Problems

Large graphs can become a ball of string, with overlapping nodes and edges. Small data changes may also cause large layout changes, disrupting users' spatial memory.

Types of Node-Link Drawings

  • Straight-line: edges are simple straight segments.
  • Polyline: edges contain several line segments.
  • Orthogonal: edges use horizontal and vertical segments.
  • Planar: crossings are avoided whenever possible.

Encoding Node and Edge Properties

NodesShape, color, size, position, and labels.
EdgesThickness, color, labels, direction, form, and curvature.

Extra encodings can communicate more information, but too many properties create clutter.

Clustering in Graphs

Clustering aggregates highly connected subgraphs to reduce complexity and reveal global structure.

many connected nodes → one cluster

The tradeoff is that aggregation may hide detailed relationships.

Graphs with Long Edge Labels

Long relationship labels create layout pressure. Drawing the label as part of the edge keeps it connected to its relationship, but the result must remain legible.

Focus and Context

Focus+context techniques enlarge an important region while keeping the surrounding structure visible. A fisheye view, for example, magnifies the focus and compresses its context.

Potential problems include curved lines, unreadable labels, unrecognizable symbols, and strong distortion near the focus boundary.

SchemeLens

SchemeLens is a topology-aware focus+context technique for large technical diagrams. Its vector-based zooming attempts to preserve orthogonal lines, alignment, and topological relationships.

Matrix Views for Graphs

An adjacency matrix places nodes along rows and columns and uses a cell mark to show a connection. Matrix views eliminate edge crossings and work well for dense graphs, although they may be less immediately intuitive than node-link diagrams.

What is a Tree?

A tree is a special graph that represents hierarchy. Each child usually has one parent, and the structure has no cycles.

Family histories File systems Organization charts Biological classification Class structures

Tree Visualizations

Node-link trees show parent-child relationships clearly for small hierarchies. As trees grow, limited space and overlapping nodes make them harder to read.

3D Tree Visualizations

AdvantagesMore layout space, animated transitions, and visual appeal.
DisadvantagesOcclusion, complexity, higher graphics requirements, and less accurate reading.

Treemaps

A treemap is a space-filling hierarchy visualization. Instead of links, it represents parent-child relationships through containment.

parent rectangle → contained child rectangles
  • Area commonly encodes a quantitative variable.
  • Color can encode a second variable.

Slice-and-Dice Treemaps

The original treemap algorithm recursively divides rectangles horizontally and vertically.

Divide parent rectangle→ Assign child rectangles→ Repeat recursively

It uses space efficiently but can create long, thin rectangles that are difficult to compare.

Problems with Treemaps

  • Areas are harder to compare than lengths.
  • Long, thin rectangles are difficult to read.
  • Labels may not fit inside small rectangles.
  • Zero values have no visible area.

Treemap Layout Algorithms

Algorithms are compared using aspect ratio, structural stability, and readability.

Slice-and-dice Cluster treemaps Squarified treemaps Strip treemaps Pivot-based treemaps

Squarified treemaps are often preferred because they reduce extreme rectangle shapes.

Map of the Market

The Map of the Market uses nested rectangles to represent stock sectors and companies. It supports:

  • Macro reading: understanding the global market structure.
  • Micro reading: inspecting individual companies.
  • Interaction: exploring details on demand.

Chapter Takeaway

Graphs visualize relationships, while trees specialize in hierarchies. Node-link diagrams are intuitive but can become cluttered; matrix views, clustering, and focus+context improve scalability. Treemaps use containment and space efficiently, but require careful layout to remain readable.
